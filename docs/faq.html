Q: What is the "Sentry" and why is it mandatory?

A: The Sentry is a Modular Verification Guard. It validates request integrity before the Brain processes any logic. It enforces a Fail-Closed security model‚Äîunauthorized traffic never touches the application state.

Q: Which Identity Standards does Nexus support?

A: Nexus is identity-agnostic. The Sentry supports pluggable modules:

  • Social/App: HMAC-SHA256 (e.g., Telegram).
  • Machine/DePIN: Ed25519 (with support for Sr25519 for peaq/Solana).
  • Federated: OIDC or DID-based resolvers.

Q: How does Nexus fit into peaq or IoTeX?

A: Nexus acts as a Sovereign Gateway.

  • For peaq: Verifies machine IDs locally before anchoring data.
  • For IoTeX: Pre-validates device intents before sending to W3bstream.
  • For TON: Manages high-frequency user interactions off-chain.

Q: Is Nexus a Bridge or an Oracle?

A: Neither. It is a Local-First Execution Layer. It does not move assets between chains; it manages local state and prepares it for global auditability via the Adapter Pattern.
