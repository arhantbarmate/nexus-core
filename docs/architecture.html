1. High-Level Design

Nexus is designed as a Unidirectional Data Pipeline. It treats the physical node as a "Sovereign Boundary" where the Sentry acts as the drawbridge.

2. Core Components

2.1 üõ°Ô∏è The Sentry (Edge Firewall)

The Sentry is a Fail-Closed middleware. It does not know what the request does; it only verifies who sent it.

  • Responsibility: Verifies cryptographic signatures (HMAC, Ed25519, Sr25519).
  • Behavior: If signature is invalid ‚Üí Drop connection immediately. No state is touched.
  • Pluggability: Supports hot-swapping auth modules (ioID, peaq ID, Telegram).

2.2 üß† The Brain (Logic Engine)

The Brain is the deterministic state machine. It executes business logic only on verified requests.

  • Isolation: Has no direct internet access; receives data only from the Sentry.
  • Reference Policy: Enforces the 60-30-10 economic split for deterministic testing.

2.3 üíæ The Vault (State & Anchoring)

The Vault is a local SQLite ledger operating in WAL (Write-Ahead Log) mode.

  • Single Source of Truth: The local database is authoritative.
  • Merkle Anchoring: The Vault aggregates state changes into a Merkle Tree. Only the Root Hash is sent to the Chain Adapter.

3. The Adapter Pattern

Nexus avoids "Vendor Lock-in" by isolating blockchain logic into Adapters. New L1/L2 integrations can be implemented with minimal code.
