Economic Protocol

Deterministic Settlement · 60/30/10 Invariant
MOD_01 // VALUE_PARTITIONING

The 60/30/10 Split Model

The protocol enforces a three-tier distribution logic for all incoming state transitions. This ensures automated, verifiable partitioning without manual reconciliation.

  • 60% Creator Share: Direct incentive for content contribution and proof-of-work.
  • 30% User Pool: Deterministic redistribution to incentivize community participation.
  • 10% Network Fee: Reserved for infrastructure maintenance and sustainability.
MOD_02 // TRUST_BOUNDARIES

Atomic Accounting

Nexus operates as a Non-Custodial ledger engine. It does not hold external assets but maintains the "Source of Truth" for distribution. All splits are calculated using integer-based primitives to prevent floating-point drift during single-transaction atomic commits.

MOD_03 // VERIFIED_INTEGRITY

The 1M Stress Test

The economic logic has been validated under extreme concurrent load:

  • Volume: 1,000,000 unique transactions validated in a single vault session.
  • Precision: Zero rounding drift observed across the entire set.
  • Throughput: Maintained strict isolation at ~50-60 TPS on edge hardware.
MOD_04 // ROADMAP

Multi-Chain Anchoring

Phase 1.4 focuses on local-first integrity. Future updates will anchor these local state roots to DePIN layers including peaq (machine identity) and IoTeX (physical activity proofs). Nexus prioritizes Durability-First local state to prevent propagation of corrupted data to global networks.
