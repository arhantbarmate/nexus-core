Technical FAQ

Sovereign Intel · Infrastructure Logic
Does Nexus require a blockchain to function?
No. Phase 1.3.1 is operational as a standalone Sovereign Node. All economic accounting and identity resolution occur within the local Brain. External chains (TON, peaq, IoTeX) are utilized in later phases for optional anchoring without affecting local execution speed or ledger correctness.
How is the Ngrok browser warning bypassed?
To maintain a $0-cost Sovereign Stack, the Brain utilizes header-based bypass logic. By injecting the ngrok-skip-browser-warning header into the outbound stream, we allow the stateless Flutter Body to establish a clean handshake without manual user intervention.
Can local hardware handle high-frequency writes?
Yes. By utilizing SQLite Write-Ahead Logging (WAL) and a durability-first engineering discipline, the Brain has been validated under a 1-Million Transaction Stress Test. It maintains stable performance (~50-60 TPS baseline) on commodity SSD hardware with zero write collisions.
Why frame the UI as a "Stateless Surface"?
To ensure Identity Agnosticism. The Body (UI) does not own state; it extracts environmental context (initData) and renders the reactive state of the Brain. This separation allows Nexus to integrate machine-IDs (ioID) or DePIN identities in Phase 2.0 without altering core logic.
Does the protocol custody user funds?
No. Nexus Protocol is Non-Custodial. The 60/30/10 split represents a deterministic accounting invariant on a local ledger. Any future integration with external settlement layers will remain opt-in, decentralized, and user-controlled.
